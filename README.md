# Tennessee Eastman Process — Fault Diagnosis with Classical ML

A multi-class fault diagnosis pipeline built on the Tennessee Eastman Process (TEP)
benchmark dataset (Downs & Vogel, 1993; simulated dataset distribution by Rieth et al.,
2017). Predicts which of 21 operating conditions (normal + 20 fault types) a chemical
process is in, using 52 sensor and actuator readings.

## Why this project

The TEP dataset is a real industrial process simulation used in actual process
monitoring and fault-detection research — not a toy dataset. Combining a chemical
engineering background with a properly validated ML pipeline (leakage-safe splitting,
grouped cross-validation, gradient boosting) makes this a strong, domain-relevant
portfolio piece.

## Results so far

| Model                | Accuracy        | Macro F1        |
|----------------------|-----------------|-----------------|
| Logistic Regression   | 0.54033        | 0.52128    |
| Random Forest         | 0.70169        | 0.72416    |
| XGBoost (untuned)     | 0.80832        | 0.81718    |

*Macro F1 is reported alongside accuracy because the 21 classes are not perfectly
balanced — accuracy alone can be misleading for multi-class problems like this.*

## Key methodology detail

Rows in this dataset are grouped by `simulationRun` — consecutive samples within a
run are highly correlated since they're the same process evolving over time. A naive
random train/test split would leak information across train and test (same run
appearing in both), silently inflating reported accuracy. This pipeline splits and
will cross-validate by `simulationRun`, so no run ever appears in both sets.

## How to run

1. Place the TEP dataset CSV in `data/raw/`.
2. `pip install -r requirements.txt`
3. Run the notebooks in order: `01_eda.ipynb` → `02_preprocessing_feature_engineering.ipynb`
   → `03_modeling_and_evaluation.ipynb`.
